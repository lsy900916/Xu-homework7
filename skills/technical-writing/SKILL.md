# 技术文档写作技能使用规范
## 使用时机
当用户需要生成技术文档、接口文档、使用手册时触发本技能。

## 写作规范
1. 标题：简洁有力，包含核心关键词，使用##/###分级
2. 格式：严格遵循Markdown，代码块必须标注语言（python/java/bash）
3. 结构：引言→核心内容→实战案例→总结/注意事项
4. 配图：每个核心概念建议添加示意图，标注图注

## 输出要求
- 包含自动生成的目录（支持[TOC]）
- 代码块需提供可运行的完整示例
- 关键步骤添加编号，注意事项用⚠️标注
